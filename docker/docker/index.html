what is docker

docker is software used for software level OS virtualization.

Container is a package containing software. More specifically, an application and allits dependencies bundled together. A regular, non-dockerised application environment is hooked directly to the OS, whereas a Docker container is an OS abstraction layer.

What Container vs docker image

If you’re familiar with object-oriented principles, another way to look at images and containers is to view images as classes and containers as objects. In the same way that objects are concrete instances of classes, containers are instances of images. You can create multiple containers from a single image, and they are all isolated from one another in the same way objects are. Whatever you change in the object, it won’t affect the class definition—they’re fundamentally different things.

Dockers commands

  • docker show all running docker docker ps ; docker container ls
  • pull nginx from repo docker pull nginx
  • see all docker images docker images
  • run docker container docker run nginx:latest
  • run container in background : detached mode
  • docker run -d nginx:latest
  • expose docker port to host:container docker run -d -p 8000:80 nginx:latest
  • stop running container docker stop id,name
  • expose multiple port docker run -p 3000:80 -p 8000:80
  • delete stopped container docker rm id
  • delete all stopped container docker rm $(docker ps -aq)
  • delete by force docker rm -f
  • give docker container name docker run --name testnameasdf
  • format docker ps output docker --format
  • attach volume to docker docker -v source_dir:docker_dir:ro
  • connect to docker interactive mode bash docker exec -it name or id bash
  • mount volume from another docker
