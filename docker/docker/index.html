Docker

what is docker

docker is software used for software level OS virtualization.

Container is a package containing software. More specifically, an application and allits dependencies bundled together. A regular, non-dockerised application environment is hooked directly to the OS, whereas a Docker container is an OS abstraction layer.

What Container vs docker image

If you’re familiar with object-oriented principles, another way to look at images and containers is to view images as classes and containers as objects. In the same way that objects are concrete instances of classes, containers are instances of images. You can create multiple containers from a single image, and they are all isolated from one another in the same way objects are. Whatever you change in the object, it won’t affect the class definition—they’re fundamentally different things.

Docker Images

  • pull nginx from repo, in this case dockerhub
    • docker image pull nginx
  • see all images in system
    • docker image ls
  • see images with sha hash
    • docker image ls --digests
  • see images by name
    • docker image ls nginx
  • remove all unused data
    • docker system prune -a -f
  • inspect docker info
    • docker image inspect fholzer/nginx-brotli
  • see docker build history of image
    • docker image history fholzer/nginx-brotli
  • full information of build of image
    • docker image history --no-trunc fholzer/nginx-brotli
  • save image with different tagname
    • docker image tag fholzer/nginx-brotli sotream/brotli:{Version}
  • docker push to repo , you have to name image your dockerhub name 'daviti' after / image name and tag to push automaticly. yet create repo at dockerhub.com
    • docker image push daviti/nginix:{Version}

Docker Containers

  • run contanier from image
    • docker run nginx
  • run container in background : detached mode
    • docker run -d nginx:latest
  • expose docker port to my host:container
    • docker run -d -p 8000:80 nginx:latest
  • use --name to name a container
    • docker run --name proxy nginx
  • keep stdin open if not attached
    • docker run -i nginx
  • allocate memory -m (10Mb (10 * 1024 * 1024))) and reservation --memory-reservation and swap
    • docker run -m 10485760 --memory-reservation 5242880 -memory-swap 10485760 nginx

  • use --restart with options to restart docker with argument

    • no - default
    • on-failure - if code in docker fails
    • always
    • unless-stopped

  • delete container after use

    • docker run --rm nginx
  • interactive mode , use bash for example
    • docker run -it nginx bash
  • see all containers
    • docker ps -s
  • from detached mode attach to container
    • docker attach id stdout stderror

  • delete all stopped container

    • docker rm $(docker ps -aq)

  • create new containter from old

CONTAINER_ID=$(docker container run -d -p 80:80 --name proxy nginx)
docker commit
--author "Full Name Email"
--message "Add curl" "$CONTAINER_ID" host/nginx-curl:0.0.1
- inspect container - docker container inspect proxy

  • delete stopped container

    • docker container rm proxy

  • delete all stopped instances

    • docker container prune
  • see docker port
    • docker container port proxy
  • see logs of container
    • docker container logs proxy
  • connect to running container

  • docker container exec -it proxy bash

  • copy files from host:container 

    docker container cp ./data/*.html proxy:/usr/share/nginx/html
docker container cp ./data/css proxy:/usr/share/nginx/html/css

  • check changes inf docker
    • docker container diff proxy

Nexus Install and push/pull image

Run Nessus

docker run -d -p 8081:8081 --name nexus sonatype/nexus3

run docker ps and grab container ID and replace below

exec to container

docker exec -it 53a6bf592078 bash

Grab the admin password

cat nexus-data/admin.password && echo

open browser to http://localhost:8081

username : admin , password :

change password and enable anonymous access

Create new repo

go to setting icon and create new repository

  • choose docker hosted
  • set name eg docker_repo
  • enable http and set port e,g 8086

get docker ip : change id in below code

docker inspect -f '{{range.NetworkSettings.Networks}}{{.IPAddress}}{{end}}' 53a6bf592078

  • in web : goto settings tab, after realms, and put docker bearer realm section to right

Commit current docker

docker commit 53a6bf592078 sonatype/nexus3:changed

Http access Linux only

search same file for windows

open /etc/docker/daemon.json file and paste , change this with your container ip and port

{
      "insecure-registries" : ["172.17.0.2:8086"]
}

restart docker

sudo systemctl restart docker

run changed nexus again

docker run -d -p 8081:8081 sonatype/nexus3:changed

**in my situation changes would not saved and i set up repo again "create new repo" section above

Login to docker registry

docker login -u admin -p 12345 http://172.17.0.2:8086/repository/docker_repo/

see docker images

Tag docker

docker tag sonatype/nexus3:latest 172.17.0.2:8086/nexus_local:latest

Push docker to local registry

docker push 172.17.0.2:8086/nexus_local:latest

Pull from local registry

docker pull 172.17.0.2:8086/nexus_local:latest
