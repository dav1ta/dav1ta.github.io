Git

Cetnralized vs Distributed Version Controls

  • Centralized - single server contains all versioned files
  • Distributed - every computer contains full history of file <- git

Hot It Stores Data: Git vs Others

  • Others - Takes files as Entry point and each commit saves only difference between old vs new version.When checkout just applies that changes.
  • Git - copies full file if there is changes. It uses file hashes , if this type of hash already is in use on another commit , git uses this version of copy of file, no duplicates will be created. Works offline fine because all changes stored locally

Git Checksumming

Every file in git repository is checksummed (SHA-1 hashed), if file changes also checksum will change and there is no way git cannot detect it.

we see that cheksums on commits too.

Git Structure

  • HEAD - Current commits value where are we at
  • Staged - Changes saved in special buffer ready for commit.
  • Worktree - single checkout version of this project. e.g if we checkout on previous commit files that are compressed in git database will uncompress and will be ready for changes.

exaple to change some settings

  amr = am --resolved
  ams = am --skip

Basic commands

  • initialize current directory
  • git init .
  • add all files to staged area
  • git add .
  • make commit from files in staged area
  • git commit -m 'message'
  • clone existing repo
  • git clone https://github.com/libgit2/libgit2
  • check changed files

  • git status

  • reset staged file

  • `git reset -- file
  • reset unstaged changes
  • git checkout -- file

Some script to see difference in realtime

myfile=$1
if [ "$1" == "" ]
then
    echo
    echo "პირველი არგუმენტი არ უნდა იყოს ცარიელი  ... გავედი"
    echo 
    exit    
fi
if [ ! -f "$1" ]
then
    echo
    echo "მოცემული $1 ფაილა არაა  ... გავედი"
    echo 
    exit    
fi
echo "HEAD:     $(git cat-file -p HEAD:$myfile) on $(git rev-parse --short HEAD)"
echo "Stage:    $(git cat-file -p :$myfile)"
echo "Worktree: $(cat $myfile)"

View Changes

  • to see what you changed locally, but it is not staged
  • git diff
  • to see staged changes
  • git diff --staged

Skipping the Staging Area

commit changes without staging changes first

git commit -a -m "commit" or git commit -am "commit

Removing Files

when u just rm files, this file will be still shown in git tracking history better to user git rm file this deletes file on computer and also in tracking history

use this command when u want remove file from git but do not delete from computer git rm --cached filename
